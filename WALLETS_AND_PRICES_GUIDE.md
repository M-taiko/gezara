# 💰 دليل استخدام الخزائن وأسعار النصيب

## الخزائن (Wallets)

### الوصول
```
http://localhost/udhiya/wallets
```

### العمليات الأساسية

#### 1. إضافة خزينة جديدة
1. انقر "➕ خزينة جديدة"
2. أدخل البيانات:
   - **الاسم:** مثل "فودافون كاش" أو "نقدي"
   - **النوع:** اختر (نقدي / محفظة رقمية / بنك)
   - **ملاحظات:** اختياري

#### 2. تحويل بين خزائن
1. في قسم "تحويل بين الخزائن"
2. اختر:
   - من الخزينة
   - إلى الخزينة
   - المبلغ
   - التاريخ
   - ملاحظات (اختياري)

#### 3. تعديل خزينة
- انقر "✏️ تعديل" على البطاقة
- غير الاسم أو النوع أو حالة التفعيل

#### 4. حذف خزينة
- **فقط** لو الرصيد = 0
- انقر "🗑️ حذف"

## أسعار النصيب في المشتريات

### الأسعار السبعة

```
1. الكامل (Full)          = السعر كاملاً
2. النصف (Half)          = نصف السعر
3. الثلث (Third)         = ثلث السعر
4. الربع (Quarter)       = ربع السعر
5. الخمس (Fifth/5)       = خمس السعر
6. السدس (Sixth/6)       = سدس السعر
7. السبع (Seventh/7)     = سبع السعر
```

### إضافة مشتريات بأسعار النصيب

#### المسار
```
Udhiya → المشتريات → ➕ مشترى جديد
```

#### الخطوات
1. اختر المورد والتاريخ
2. **أضف سطر (صنف):**
   - اختر المنتج (نوع الحيوان)
   - الكمية (1 لكل حيوان)
   - الوزن (kg، اختياري)
   - التكلفة الكلية
   - **جديد:** 7 أعمدة للأسعار (املأ اللي تحتاجه)
3. **اختر الخزينة** للدفعة الأولى (اختياري)
4. احفظ

**الحيوانات تُنشأ تلقائياً بنفس الأسعار** من purchase_items

#### مثال
```
المنتج: بقرة عربي
الكمية: 1
التكلفة: 50,000 ج.م

الأسعار:
- الكامل: 50,000
- النصف: 25,000
- الثلث: 16,667
- الربع: 12,500
- الخمس: 10,000
- السدس: 8,333
- السبع: 7,143
```

### تعديل مشتريات

#### المسار
```
Udhiya → المشتريات → اختر الفاتورة → ✏️ تعديل الفاتورة
```

#### التعديل
- يمكن تغيير **الأسعار** مباشرة في الأعمدة
- الحيوانات **لا تتأثر** بالتعديل (الأسعار فقط تُحفظ في purchase_items)

## ربط الخزائن بالدفعات

### تسجيل دفعة من عميل

#### المسار
```
Udhiya → التقارير → كشف حساب عميل
```

#### النموذج الجديد
```
الصك:        [اختر]
المبلغ:      [أدخل]
الطريقة:     [نقدي / بنك / تحويل]
🆕 الخزينة:  [اختر: فودافون كاش / نقدي / ...]
التاريخ:     [YYYY-MM-DD]
```

**عند الحفظ:**
- ✅ الدفعة تُسجل مع wallet_id
- ✅ رصيد الخزينة يزيد تلقائياً
- ✅ المعاملة تُسجل في سجل الخزينة

### تسجيل دفعة للمورد

#### المسار
```
Udhiya → التقارير → كشف حساب مورد (أو صفحة الفاتورة)
```

#### النموذج الجديد
```
الفاتورة:    [اختر]
المبلغ:      [أدخل، الحد الأقصى = المتبقي]
🆕 الخزينة: [اختر: الخزينة المخصومة]
التاريخ:     [YYYY-MM-DD]
ملاحظات:    [اختياري]
```

**عند الحفظ:**
- ✅ الدفعة تُسجل مع wallet_id
- ✅ رصيد الخزينة ينقص تلقائياً
- ✅ المعاملة تُسجل في سجل الخزينة

---

## التقارير والسجلات

### رصيد الخزينة الفوري
```
الخزائن → بطاقات
```
- يظهر الرصيد الفعلي (يتحدث مع كل دفعة)
- آخر 3 معاملات على الخزينة

### سجل المعاملات
```
الخزائن → بطاقة الخزينة → آخر المعاملات
```
- تاريخ → مبلغ ← نوع (دخول/خروج)

## ملاحظات مهمة

### ✅ ما يشتغل
- الخزائن بدون wallet_id: ما في مشكلة
- wallet_id optional في كل الدفعات
- يمكن يومين دفع الأول بخزينة والثاني بدون

### ⚠️ قيود مقصودة
- **حذف خزينة:** فقط لو الرصيد = 0
- **تحويل:** تحقق من رصيد كافي قبل التحويل
- **دفع:** لا تقدر تدفع أكثر من المتبقي

### 🔍 إذا حصلت مشكلة
```bash
# تحقق من الأرصدة
php artisan tinker
>>> App\Models\Wallet::all()->pluck('name', 'balance');

# شوف آخر معاملات
>>> App\Models\WalletTransaction::latest()->take(5)->get();

# شوف آخر دفعات
>>> App\Models\Payment::with('wallet')->latest()->take(5)->get();
```

## أمثلة عملية

### مثال 1: عميل يدفع صك بفودافون كاش
```
1. كشف حساب العميل → نموذج الدفعة
2. اختر الصك
3. أدخل المبلغ (مثلاً 10,000)
4. اختر الطريقة: نقدي
5. اختر الخزينة: 💰 فودافون كاش
6. اضغط ✅ تسجيل الدفعة

النتيجة:
✓ الدفعة مسجلة
✓ رصيد "فودافون كاش" زاد 10,000
✓ الصك قل المتبقي بـ 10,000
```

### مثال 2: شراء من مورد بأسعار نصيب مختلفة
```
1. مشتريات → مشترى جديد
2. اختر المورد
3. أضف صنف:
   - المنتج: بقرة بلدي
   - الكمية: 1
   - الوزن: 350 كجم
   - التكلفة: 50,000
   - الكامل: 50,000
   - النصف: 25,000
   - الثلث: 16,700
   - الربع: 12,500
   - ... (باقي الأسعار)
4. اختر الخزينة: نقدي
5. احفظ

النتيجة:
✓ حيوان واحد منشأ بهذه الأسعار
✓ لو اختر عميل "نصف" من هذا الحيوان = 25,000
✓ لو اختر عميل "ثلث" = 16,700
```

### مثال 3: دفع للمورد من خزينة معينة
```
1. صفحة الفاتورة → 💰 تسجيل دفعة
2. المبلغ: 20,000 (متبقي 30,000)
3. الخزينة: 🏦 بنك
4. التاريخ: اليوم
5. احفظ

النتيجة:
✓ 20,000 انقصت من رصيد "بنك"
✓ الفاتورة بقي فيها 10,000 متبقي
✓ سجل المعاملة في الخزينة: -20,000 (خروج)
```

## الأسئلة الشائعة

**س: هل يمكن إنشاء مشتريات بدون أسعار نصيب؟**  
**ج:** نعم، الأسعار اختياري. لو ما حددت حاجة تترك فارغة.

**س: هل يمكن تحويل فلوس بين خزينتين من صفحة الخزينة؟**  
**ج:** نعم، قسم "تحويل بين الخزائن" في نفس الصفحة.

**س: لو سجلت دفعة من عميل بدون خزينة، هل فيه مشكلة؟**  
**ج:** لا، الدفعة تسجل بدون مشكلة، بس ما يزيد رصيد خزينة.

**س: هل يمكن حذف خزينة فيها معاملات؟**  
**ج:** لا، فقط لو الرصيد = 0. لازم تحويل أو إنفاق الفلوس الأول.

**س: إذا تعديل الأسعار في purchase، الحيوانات تتأثر؟**  
**ج:** لا، الأسعار تُحفظ في الحيوان مرة واحدة عند الإنشاء. التعديل لاحقاً ما يأثر على الحيوانات.
