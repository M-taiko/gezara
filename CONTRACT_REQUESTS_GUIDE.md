## مقدمة

نظام طلبات الاشتراك يسمح للعملاء بالتقديم على نصيب من حيوان مباشرة من الصفحة الرئيسية (الداشبورد) دون الحاجة لإنشاء حساب أو تسجيل بيانات معقدة.

### المميزات
- ✅ واجهة سهلة من الصفحة الرئيسية
- ✅ عرض جميع الحيوانات المتاحة بطريقة بطاقات
- ✅ تمييز بين الحيوانات المجمعة والمستقلة
- ✅ عرض الأنصبة المتبقية لكل حيوان مجمع
- ✅ جمع بيانات العميل في modal سهل
- ✅ لوحة تحكم كاملة لإدارة الطلبات

## كيفية التقديم على نصيب

### المسار
```
الصفحة الرئيسية → الحيوانات المتاحة → اختر حيوان → اضغط "تقديم على نصيب"
```

### الخطوات

#### 1. من الصفحة الرئيسية
1. اذهب إلى **لوحة التحكم** (الرابط الأول في الـ sidebar)
2. ستظهر **"الحيوانات المتاحة"** في قسم خاص

#### 2. اختر حيوان
كل بطاقة حيوان تعرض:
- 🐄 **نوع الحيوان**: مثل "عجل بلدي" أو "بقرة عربي"
- 📍 **الكود**: رقم تعريفي فريد (مثل 2026-BQR-0001)
- 💰 **السعر الكامل**: السعر الكلي للحيوان الواحد
- 🔗 **الحالة**: "مجمع" (بأنصبة) أو "مستقل" (كامل)
- إذا كان **مجمعاً**:
  - 📍 **نوع النصيب**: السبع، السدس، الخمس، الربع، الثلث، النصف
  - 📊 **الأنصبة المتبقية**: عدد الأنصبة المتاح حجزها

#### 3. اضغط على الزر
- إذا كان هناك أنصبة متبقية:
  - اضغط 🎯 **"تقديم على نصيب"**
- إذا امتلأ:
  - الزر يكون معطل ❌ **"غير متاح"**

#### 4. ملء بيانات التقديم
النموذج يطلب:

```
👤 اسم العميل *               (مثل: محمد أحمد)
📱 رقم الهاتف *               (مثل: 01234567890)
📧 البريد الإلكتروني           (اختياري - مثل: email@example.com)
💰 السعر المتوقع              (اختياري - يترك فارغاً للتحديد لاحقاً)
📝 ملاحظات                    (اختياري)
```

**الحقول المطلوبة:**
- ✅ اسم العميل
- ✅ رقم الهاتف
- ℹ️ البريد والسعر والملاحظات: اختياري

#### 5. إرسال الطلب
- اضغط ✅ **"إرسال الطلب"**
- سيظهر إشعار: "✅ تم استلام طلب الاشتراك بنجاح، سيتم التواصل معك قريباً"
- سيتم حفظ الطلب وسيصل إلى **"طلبات الاشتراك"** في الداشبورد

## إدارة الطلبات

### رابط الدخول
```
الـ Sidebar → 📋 طلبات الاشتراك
أو: http://localhost/udhiya/contract-requests
```

### واجهة الإدارة

#### أعلى الجدول
```
عدد الطلبات: [الإجمالي]

فلتر الحالة:
  [ بحث ▼ ]  [ 🔍 البحث ]  [ 🔄 إعادة تعيين ]
```

#### الحالات المتاحة في الفلتر
- ⏳ **معلقة** (pending) — الطلبات الجديدة
- ✅ **موافق عليها** (approved) — الطلبات المقبولة
- ❌ **مرفوضة** (rejected) — الطلبات المرفوضة
- 📋 **تحويل لصك** (converted) — الطلبات المحولة لصكوك فعلية

### الإجراءات على الطلبات

#### للطلبات المعلقة
```
✅  — اضغط لقبول الطلب
❌  — اضغط لرفض الطلب
```

عند الاختيار:
1. سيظهر تأكيد: "هل أنت متأكد؟"
2. بعد التأكيد: سيتغير الحالة فوراً
3. الصفحة ستُحدّث تلقائياً

#### للطلبات المقبولة
```
📋  — اضغط لتحويل الطلب إلى صك فعلي
```

عند الضغط:
1. سيتم تحويل الطلب إلى صك (contract) حقيقي
2. سيتم ربط العميل والحيوان والنصيب
3. ستتغير حالة الطلب إلى "تحويل لصك"

## البيانات المخزنة

### جدول `contract_requests`
```sql
id                  — معرف الطلب
animal_id           — معرف الحيوان (FK)
customer_name       — اسم العميل (مثل: محمد أحمد)
customer_phone      — رقم هاتف العميل
customer_email      — بريد العميل (اختياري)
share_type          — نوع النصيب (seven, six, five, quarter, third, half, full)
share_price         — السعر المتفق عليه (اختياري)
notes               — ملاحظات العميل (اختياري)
status              — الحالة (pending, approved, rejected, converted)
created_at          — تاريخ الطلب
updated_at          — آخر تحديث
```

### مثال من البيانات
```
ID: 5
Animal: 2026-BQR-0003 (عجل بلدي)
Customer: محمد أحمد
Phone: 01234567890
Email: mohammed@example.com
Share: half (النصف)
Price: 25,000 ج.م
Status: ⏳ معلقة
Created: 2026-04-13 14:35:22
```

## الحالات والمسارات

### دورة حياة الطلب

```
┌─────────────────────────────────────────────────┐
│  1️⃣ العميل يقدم طلب من الصفحة الرئيسية         │
│     ↓                                            │
│  حالة: ⏳ معلقة (pending)                      │
│     ↓                                            │
│  2️⃣ المدير يراجع الطلب في لوحة التحكم          │
│     ↙         ↘                                  │
│  قبول    أو    رفض                            │
│   ↓               ↓                              │
│  ✅ موافق    ❌ مرفوض                        │
│  عليها         (نهاية)                         │
│   ↓                                             │
│  3️⃣ تحويل إلى صك                             │
│   ↓                                             │
│  📋 تحويل لصك                                 │
│  (نهاية - صك حقيقي)                           │
└─────────────────────────────────────────────────┘
```

### الانتقالات المسموحة

| من | إلى | الزر |
|----|-----|------|
| معلقة | موافق عليها | ✅ |
| معلقة | مرفوضة | ❌ |
| موافق عليها | تحويل لصك | 📋 |

## أمثلة عملية

### مثال 1: تقديم على نصف حيوان

```
1. من الصفحة الرئيسية
2. ترى بطاقة:
   - 🐄 عجل بلدي
   - كود: 2026-BQR-0001
   - سعر: 50,000 ج.م
   - نوع: 🔗 مجمع
   - نصيب: النصف (2 أنصبة)
   - متبقي: 1 نصيب
3. اضغط 🎯 "تقديم على نصيب"
4. املي:
   - اسم: أحمد محمد
   - هاتف: 01098765432
   - بريد: ahmed@example.com
   - سعر: 25,000
   - ملاحظة: أريد الدفع بالتحويل البنكي
5. اضغط ✅ "إرسال الطلب"
6. الطلب وصل → يظهر في لوحة التحكم
```

### مثال 2: قبول وتحويل الطلب

```
1. اذهب إلى 📋 طلبات الاشتراك
2. تجد الطلب من أحمد محمد (معلق)
3. اضغط ✅ في نفس الصف
4. تأكد: "هل أنت متأكد؟"
5. الطلب صار: ✅ موافق عليها
6. اضغط 📋 في نفس الصف
7. الطلب انحول: 📋 تحويل لصك
8. الآن تجد صك حقيقي في قائمة الصكوك
```

## ملاحظات مهمة

### ✅ ما يشتغل
- تقديم متعدد من نفس العميل على أنصبة مختلفة
- تقديم على نصيب في حيوانات مختلفة
- التعديل على البيانات بعد التقديم (من خلال المدير)

### ⚠️ قيود مقصودة
- **فقط الأنصبة المجمعة**: لا يمكن التقديم على حيوان مستقل (غير مجمع)
- **أنصبة محدودة**: لا تستطيع التقديم إذا انتهت الأنصبة
- **لا يمكن حذف**: الطلب المحول لصك لا يمكن حذفه

### 🔍 إذا حصلت مشكلة

```bash
# شوف كل الطلبات
php artisan tinker
>>> App\Models\ContractRequest::all();

# شوف طلب معين
>>> App\Models\ContractRequest::find(5);

# اعدل حالة طلب
>>> $req = App\Models\ContractRequest::find(5);
>>> $req->status = 'approved';
>>> $req->save();

# احذف طلب (بحذر!)
>>> App\Models\ContractRequest::find(5)->delete();
```
